# Antigravity 에이전트 개발 규칙 (dobby-project)

이 문서는 경제적 자유 프로젝트(`dobby-project`)를 위한 개발 원칙 및 AI 에이전트 전용 행동 강령입니다. 에이전트는 코드 작성 시 이 규칙을 엄격하게 준수해야 합니다.

## 1. 프로젝트 핵심 가치
* **안전성 최우선**: 주식 자동매매 시스템 개발 시, 실제 자산이 투입되므로 비정상적인 거래나 예외 상황에 대한 방어 코드를 철저히 작성해야 합니다. 모의투자 모드를 기본값으로 제공합니다.
* **사용자 친화적 대시보드**: 자산 대시보드는 비주얼 완성도(풍부한 미학, 반응형 차트)를 높게 유지합니다.

## 2. 기술 스택 가이드라인
* **Front-end**:
  * React (TypeScript) + Vite 환경을 기본으로 합니다.
  * 스타일링은 TailwindCSS 또는 Vanilla CSS를 깔끔하게 사용합니다.
  * 차트 시각화 라이브러리로 `recharts` 또는 `chart.js`를 사용합니다.
* **Back-end & Trading**:
  * Python 3.9+ 버전을 기준으로 작성합니다.
  * 한국투자증권 API(KIS API) 등 국내 증권사 API를 우선 적용합니다.
  * 민감정보(AppKey, SecretKey 등)는 절대 코드에 하드코딩하지 않고 `.env` 파일에 보관합니다.

## 3. 코드 작성 규칙
* **주석 및 설명**: 한국어로 변수명과 함수 기능에 대한 명확한 주석을 달아줍니다.
* **예외 처리**: 매매 로직 실행 중 API 호출 실패나 네트워크 오류 시 대기 시간 부여(Retry) 및 카카오톡/텔레그램 알림을 발송하는 로직을 필수로 구성합니다.

## 4. 실거래 정산 및 대시보드 회계 규칙
* **토스증권 해외주식 수수료 기준 적용**: 토스증권 API 연동 봇 구동 및 가상 정산 시, 수수료는 반드시 토스증권 실거래 기본 기준인 **`0.1%` (0.001)** 로 통일하여 연산합니다. (이벤트 우대 수수료 등 임의 요율 사용 금지)
* **미국 매도 제세금 (SEC Fee) 동기화**: 미국 시장 주식 매도 시 차감하는 제세금은 미국 증권거래위원회 고시 요율을 상시 모니터링하여 최신화합니다. (2026년 4월 2일 고시 요율 기준 **`0.00206%`** 적용)
* **자산 공식의 투명화 및 계정 분리**:
  * 자산 계산 시 평단가나 매수량으로 현금을 유추(역산)하는 공식을 사용해서는 안 되며, 반드시 세전 현금(`cash`)과 세후 현금(`net_cash`) 잔고를 개별 변수로 나누어 1원 단위까지 독립 누적 추적해야 합니다.
  * 웹 대시보드 내 모의투자 현황은 반드시 **`예수금(보유 현금)`**, **`주식 평가금액`**, **`주식 평가손익(수익률%)`**, **`가상 총자산`**으로 개별 분할 표기하여 금융 전문성을 보장합니다.
* **모바일 크롬 파비콘 캐시 무력화**: 모바일 브라우저의 파비콘 갱신 지연 현상을 방지하기 위해 헤더 영역에 명시적인 투명 1x1 base64 인라인 아이콘 태그(`data:;base64,...`)를 선언하여 이전 파비콘 캐시 흔적을 강제 무력화해야 합니다.
